#  Project: Order summary component (Frontend Mentor Challenge)

Welcome to your first challenge! The goal is to accurately reproduce the provided design while following modern web development best practices.

##  Mentoring Objectives
- **Design Precision:** Match colors, spacing, and fonts from the `style-guide.md`.
- **Clean Code:** Use semantic HTML5 and clear CSS class names.
- **Responsive Design:** The profile must look perfect on both Mobile  and Desktop.
- **Git Flow:** Make regular commits with clear messages with issue number(e.g., `feat: #5 add hover states to buttons`).

---

##  Startup Procedure (Step-by-Step)

Follow and reproduce this procedure on your local environment to get started:

### 1. Repository Setup
- [ ] **Clone:** Open your terminal and clone this repo: `git clone [REPO-URL]`
- [ ] **Explore:** Move into the folder: `cd order-summary-component`

### 2. Get Frontend Mentor Resources
- [ ] **Download:** Go to the [Challenge Link](https://www.frontendmentor.io/challenges/order-summary-component-QlPmajDUj), click "Start Challenge," then "Download starter code."
- [ ] **Extract:** Unzip the content directly into your local project folder.
- [ ] **Clean up:** You can delete or rename the `README-template.md` provided by FM to keep only this one.

### 3. Initial Synchronization (First Push)
- [ ] **Stage:** Run `git add .` to prepare the starter files.
- [ ] **Commit:** Run `git commit -m "chore: initial starter files from Frontend Mentor"`
- [ ] **Push:** Send everything to GitHub with `git push origin main`.

### 4. Design Analysis
- [ ] Open `style-guide.md` and set up your CSS variables in a `style.css` file.
- [ ] Study the screenshots in the `/design` folder before writing your first line of code.
